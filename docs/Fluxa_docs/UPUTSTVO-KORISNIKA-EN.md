Fluxa User Manual
1. Introduction
Fluxa is an operating system that connects the entire workflow of an agency�from the first client contact to the final invoice and calculation of the actual project margin.
Fluxa helps agencies turn structured deals into controlled projects and profitable invoices, providing clear visibility of costs, team work, and final profit.
The basic workflow in Fluxa is simple:
Deal ? Project ? Invoice ? Profit
The process works as follows:
1. A deal is created and the project scope is defined.
2. After client approval, the deal becomes a project where the team tracks work stages and costs.
3. When the work is complete, an invoice is issued.
4. Fluxa automatically calculates the actual project margin.
The system aims to give agencies a clear view of how profit is generated and where costs occur.

1. What is Fluxa
Fluxa is an operational control system designed for creative and marketing agencies.
The system connects the entire agency work lifecycle in a single clear flow:
Deal ? Project ? Invoice ? Actual Profit
Fluxa enables owners and managers to:
* Plan projects
* Track actual work costs
* Control budgets
* Analyze project profitability
Instead of using multiple separate tools for sales, projects, and finance, Fluxa combines everything into a single operational system.

2. Who Fluxa is For
Fluxa is designed for small and medium agencies that manage multiple client projects simultaneously.
Typical users include:
* Marketing agencies
* Creative studios
* Branding agencies
* Digital production agencies
* Design and communication studios
It is especially useful for:
* Agency owners
* Project managers
* Account managers
Fluxa provides a clear overview of projects, costs, and profitability without the need for complex business software.

3. How Fluxa Works
Fluxa organizes work through three interconnected phases:
1. Deals
In this phase, a potential project is defined:
* Client
* Scope of work
* Estimated budget
2. Project
When the client approves the deal, it becomes a project. During the project, Fluxa tracks:
* Work progress
* Project phases
* Internal costs
3. Invoice
After the project is completed:
* An invoice is issued
* The system calculates the actual project margin
This workflow gives agency owners a complete picture of business operations.

3. Deals (Negotiations)
The Deals module manages sales opportunities before a deal becomes a project.
Each deal represents a potential client project. At this stage, you define:
* Deal name
* Client
* Estimated budget
* Responsible person
During negotiations, status can be changed to track progress. Each deal has a timeline for:
* Notes
* Offer changes
* Client communications
When the client accepts the offer, the deal becomes a project.

Deal Details
The deal window is the central place to manage negotiations. Here you define:
* Project deadlines
* Budget
* Offer items
* Negotiation workflow

Deal Status and Phases
At the top is a status bar showing the negotiation phase:
* Deal � initial phase
* Production � project accepted
* Completed � project finished
* Closed � administratively closed
* Invoiced � invoice issued
* Archived � project archived
Once closed, the deal becomes read-only.

Timeline
The timeline records key negotiation events:
* Deal opening date
* Project deadline
* Deal confirmation method
* Notes

Items and Budget
In the Items section, financial aspects are defined. Users can:
1. Select an item from the price list
2. Define quantity
3. Enter price
4. Add a description
The system automatically calculates:
* Item value
* Total project budget

Converting a Deal to a Project
When the client confirms the deal:
* Budget transfers to the project
* Project workflow starts
* Negotiation details are locked
The deal then becomes the basis for operational project work.

4. Project Overview (PP)
The Project Overview shows all projects in the system.
Here you can:
* View active projects
* Filter by status
* Review project archives
* Sort projects by criteria
From this list, you can open project details and manage phases and finances.

5. Project Details
The Project Details screen is the central operational console. Its main purpose is to connect planned budgets with actual costs, allowing project managers to monitor profitability at all times.
Here you manage:
* Project status
* Financial costs
* Work phases
* Project completion

Status and Workflow
At the top is the workflow bar showing current project status. Changing status signals the system which phase the project is in.
Example phases:
* Production
* Completed
* Closed
* Invoiced
* Archived
When the status is Invoiced, the system prepares for financial closure and billing.

Tracking Financial Status
Fluxa automatically tracks project financials in real time, showing:
* Planned budget
* Total costs
* Expected profit
Project managers aim to keep costs below budget to ensure a positive project margin.
Visual indicators (colors and status) quickly show if the project exceeds financial limits.

Operational Cost Entry
The cost entry section records all expenses incurred during the project. Each cost entry defines:
* Type (e.g., fee, service, production)
* Person/collaborator responsible
* Amount
* Currency
If marked as incurred, the system deducts it from the project budget; if planned, it reserves the budget.

Currency Conversion
Fluxa supports multi-currency cost entry. The Rate field automatically converts expenses to the system base currency using the Central Bank rate at the time of entry.

Documentation and Audit
Each expense can include a note explaining context. Notes help with:
* Financial audits
* Project analysis
* Internal communication
The cost table at the bottom shows a chronological record of entries.

Project Phase Planning
The PHASES button opens a Gantt chart displaying all project phases and deadlines.
This allows:
* Planning phase order
* Defining micro-deadlines
* Controlling dependencies
Fluxa prevents phase deadlines from exceeding the overall project deadline.

Project Completion
The FINAL OK button is used when all phases are complete. At this point:
* Project is marked complete
* Returned to the Deal module
* Account manager confirms completion with the client
When the client confirms, status changes to Closed. The project becomes read-only and moves to the financial module for invoicing.

6. Invoicing
Fluxa uses a three-step wizard to generate invoices. This converts completed projects into official financial documents.
Step 1 � Project Selection
Select projects to include in the invoice. Filter by:
* Client
* Time period
* Project status
Closed projects are ready for invoicing. The system displays:
* Base price
* VAT amount
* Total amount
Step 2 � Invoice Settings
Define invoice parameters:
* Invoice date
* Currency
* VAT regime
* Project/item name
The system generates a reference number for automatic payment matching. Project name can be adjusted to match client terminology.
Step 3 � Invoice Preview
Displays a visual preview before issuance, showing:
* Issuer data
* Client data
* Invoice items
* VAT
* Total amount due
The system adds bank payment instructions. The invoice can then be downloaded as PDF or sent to the client.

7. Payments
Fluxa supports automatic payment recording via bank statements in XML V2 format. Upon import, the system:
* Recognizes the reference number
* Matches payment to invoice
* Marks invoice as paid
